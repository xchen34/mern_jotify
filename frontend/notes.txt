名字	类型	导入方式	为什么？
Toaster	组件 (Component)	import { Toaster }	它是库里的一个特定工具。
toast	函数 (Function)	import toast	它是库的默认核心功能。
App	组件 (Component)	import App	因为你写了 export default App。

具名导出（Named Exports）与默认导出（Default Exports）的区别，以及组件（Component）与函数（Function）**的区别。
A. 具名导出 (Named Export) → 必须打 { }

库的代码里写的是 export const Toaster = ...。

    这种方式允许一个文件导出很多个东西。

    你引用时必须点名道姓，且名字必须用大括号包起来。

    例子：import { Toaster, useToaster } from "react-hot-toast"。

B. 默认导出 (Default Export) → 不打 { }

库的代码里写的是 export default toast。

    一个文件只能有一个默认导出。

    你引用时可以随便起名字（虽然通常还是叫 toast），且不需要大括号。

    例子：import myToast from "react-hot-toast"（这依然能指向那个函数）

     一般默认导出 (Default Export)：通常是该文件的“主角”。一个文件只能有一个主角。

    语法： import App from './App'

具名导出 (Named Export)：通常是该文件的“配角”或“工具包”。一个文件可以有很多配角。

    语法： import { Toaster, toast } from 'react-hot-toast'

    React 核心：import React, { useState, useEffect } from 'react' (React 是主角，Hook 是配角)。

    React Router：全都是配角 import { Routes, Route, Link } from 'react-router-dom'


    2. Component (组件) 就是标签吗？

答案是：是的，但它是“自定义”的超级标签。

    HTML 标签：比如 <div>, <h1>, <button>。它们是浏览器自带的，全部是小写字母开头。

    React 组件：比如 <App />, <Toaster />, <HomePage />。它们是你或者库作者写的 JavaScript 函数，必须以大写字母开头。

为什么叫“超级标签”？

因为 HTML 标签只能展示简单的内容，而 Component (组件) 内部可以包含：

    自己的逻辑（比如点击后算数）。

    自己的状态（比如记住你输入了什么）。

    其他标签（一个组件可以包裹成千上万个 div）。

3. Toaster 组件就是写成 <Toaster /> 吗？

没错！

在 React 里，只要一个函数返回的是 JSX（看起来像 HTML 的代码），它就是一个组件。你可以像使用 HTML 标签一样使用它：

    如果它不需要包裹内容：写成自闭合 <Toaster />。

    如果它需要包裹内容：写成 <BrowserRouter> ... </BrowserRouter>。

dist 是 distribution（分发） 的缩写，是前端项目打包后生成的生产环境代码文件夹。

为什么需要打包？
开发时你写的代码（在 src 目录）：

使用 JSX 语法（浏览器不认识）
使用 ES6+ 模块 (import/export)
使用 Tailwind CSS 等需要编译的工具
代码分散在多个文件中
浏览器无法直接运行这些代码！

npm run build 做了什么？
打包工具（你的项目用的是 Vite）会：

编译转换：JSX → 纯 JavaScript，Tailwind → 标准 CSS
模块打包：将所有 .jsx、.js、.css 文件合并压缩
代码优化：压缩代码体积，删除注释和空格
生成静态文件：最终输出到 dist/ 文件夹
dist/ 文件夹内容示例：
dist/
├── index.html          # 入口HTML文件
├── assets/
│   ├── index-abc123.js   # 打包后的所有JS代码（带hash）
│   ├── index-def456.css  # 打包后的所有CSS代码
│   └── logo-xyz789.png   # 静态资源
最终流程：
开发：npm run dev → Vite 开发服务器（5173端口）
打包：npm run build → 生成 dist/ 文件夹
部署：后端服务器通过 express.static() 托管 dist/ 文件夹
访问：用户访问 http://localhost:5001/ → 获取 dist/index.html 及其资源
这就是为什么你的后端能直接提供前端页面的原因！
